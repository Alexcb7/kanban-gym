He organizado el proyecto separando la lógica del estado y la parte visual. Todo el estado del tablero se gestiona desde un hook (useBoard) y los componentes solo se encargan de mostrar la información. Para la auditoría decidí guardar solo los campos que cambian (diff before/after) en vez de la tarea completa, así el registro es más claro y ligero. La búsqueda avanzada funciona con un parser propio que convierte el texto en reglas y genera un filtro dinámico. No se usa base de datos, así que todo se guarda en localStorage, incluyendo validación cuando se importa un JSON. El Drag & Drop está hecho con @dnd-kit, intenté utilizar el Drag Overlay para que fuera más sútil y fino pero me acabo dando errores a la hora de coger y arrastrar y separé la lógica de mover y reordenar tareas para que el código sea más limpio y fácil de mantener.
